## 阶段2：模式发现

### 论证结构模式（80%+文章共用同一模板）

通过对100篇随机样本的结构标注，发现**固定三段式**：

```
[钩子] 一个反常的具体案例/场景（让读者觉得「诶？」）
  ↓
[拆解] 揭示案例背后的实际规则（「你以为的是A，其实运作的是B」）
  ↓
[泛化] 将规则提升为普遍原则（「不只是这里，X和Y也一样」）
  ↓
[金句] 一句可以单独截图的结论
```

**变体1：对比型**（约20%文章中）
```
场景A（外国的失败做法）→ 场景B（中国的做法）→ 规则差异 → 结论
例：日本躺平 vs 中国不该躺平
```

**变体2：反转型**（约15%文章中）
```
常识观点 → 为什么常识是错的 → 正确观点 → 结论
例：越穷越该稳定（常识）→ 稳定就是等死 → 越穷越要折腾
```

### 高频概念网络

从376篇中提取高频共现概念：

```
规则 ←→ 成本 ←→ 基本盘 ←→ 差异化
  ↑       ↑        ↑
  │       │        │
本质    转嫁    框架
  ↑       ↑        ↑
  └───────┴────────┘
      战略选择
```

**核心发现**：鹤老师的思维是一个以「规则」为根节点、辐射到「成本/基本盘/差异化/框架」的树状结构。「规则」是他最底层的透镜。

### 类比来源分类

随机抽取50个类比，分类来源：

| 类比来源 | 占比 | 典型例子 |
|---------|------|---------|
| 日常生活场景 | 35% | 金鱼不能只养一只、晚高峰共享单车、酒店前台欢迎光临 |
| 商业/互联网案例 | 30% | 段永平/黄铮、拼多多、淘宝、英伟达CUDA |
| 经济学原理 | 15% | 科斯定理→追尾全责、杰文斯悖论→芯片需求 |
| 动物/自然现象 | 10% | 骆驼挤帐篷、鱼不知道水的存在 |
| 历史/典故 | 5% | 二战经济学家、德国眼镜蛇困局 |

**核心发现**：他的类比高度依赖**大众日常经验**（35%+30%商业案例=65%的人都知道的事），极少用学术或小众引用。这使得他的文章「谁都能看懂」。

## 阶段4：操作系统逆向 ⭐（核心产出）

### 4.1 选题策略：他怎么决定「写什么」

**规则1：反向扫描法**
- 不自己想话题 → 看大家在**焦虑什么、争论什么、误解什么**
- 选题来源：社交媒体的高频争议、大众常识中的「对但错」、新闻热点中的「隐藏规则」
- 决策链：这是大众关心的话题？→ 这里面有没有一个被误解的规则？→ 三句话能说清楚吗？→ 能 → 写

**规则2：可泛化性测试**
- 问自己：这个案例只在这件事上成立，还是一类事都成立？
- 「长租公寓爆雷」→ 不只是公寓，是「所有资金盘生意」→ 泛化成功 ✅
- 「700万房子买错」→ 不只是买房，是「所有信息不对称的交易」→ 泛化成功 ✅
- 泛化不了的 → 不写

**规则3：反常识检测**
- 这个话题的「常识说法」是什么？→ 这个常识有什么漏洞？→ 漏洞能推出一个相反结论吗？
- 能 → 有冲突 → 有阅读价值 → 写
- 不能 → 太平淡 → 不写

**规则4：可节省性测试**
- 这篇文章能不能浓缩成一句金句？→ 能 → 说明核心洞察清晰 → 写
- 不能 → 说明自己也没想清楚 → 回到规则1重新选

### 4.2 论证构建：他怎么从「一个想法」走到「一篇文章」

**标准操作流程：**

```
Step 1: 找钩子
- 选一个具体的、反常的、读者能立刻代入的案例
- 钩子标准：1)真实发生过 2)读者可能经历过 3)第一反应是A但结果是B
- 例：朋友700万买房、瑞士金鱼要养两只、长租公寓爆雷

Step 2: 映射到规则
- 这个案例的「表面逻辑」是什么？「实际逻辑」是什么？
- 实际逻辑=规则。用最日常的话说出来，不要用术语。
- 例：长租公寓→「不是出租生意，是资金盘生意」

Step 3: 建立泛化
- 不只这个案例，X和Y也是同样的逻辑
- 连接2-3个不同领域的案例 → 证明这不是特例，是普遍规则
- 例：长租公寓→互联网补贴→免费送房→都一样→「成本不会消失，只会转嫁」

Step 4: 给出操作建议
- 基于这个规则，读者应该怎么做？
- 建议必须是可操作的，不能是「要保持警惕」这种废话
- 例：买房→「翻完所有APP评论区」

Step 5: 收尾金句
- 把核心洞察压缩成15字以内的一句话
- 必须带情绪/态度/立场，不能中性
- 例：「挺直腰板，别老跪着做人」「成本不会消失，它只会转嫁到弱势的那一方」
```

### 4.3 类比生成机制

**他的类比生成偏好**：

| 维度 | 偏好 | 为什么 |
|------|------|--------|
| **来源域** | 日常可见的现象（动物、超市、停车场） | 降低理解门槛——不用读者有背景知识 |
| **映射方式** | 结构相似而非表面相似 | 他找的不是「长得像」，是「运作逻辑像」 |
| **情感色调** | 轻微荒诞/讽刺 | 让类比不仅解释问题，还制造记忆点 |
| **复杂度** | 一句话能说完的类比 | 长类比=读者注意力流失 |

**类比生成模板**（可学习用于生成新的类比）：

```
模板：把 [复杂规则] 类比为 [日常现象]
条件：两者的运作逻辑结构相同，但日常现象人人都见过
调性：轻微荒诞，让人觉得「好像有点道理但又有点好笑」

举例：
- [边界扩大的危险] → [骆驼挤帐篷]
- [表面看专业实际看基本盘] → [星光大道干翻经济半小时]
- [数据维度不可比] → [500克和200米哪个大]
```

### 4.4 反直觉推导链

**他的反直觉推导，几乎都指向同一个根因：表面规则 ≠ 实际规则**

**推导模板**：

```
常识：大家说X是对的
 ↓
反问：X背后的规则是谁定的？谁在受益？
 ↓
发现：定规则的人讲的故事 ≠ 实际运行的逻辑
 ↓
结论：X可能恰恰是错的，反着来的Y才更接近真相
```

**实例化**：

| 常识 | 拆解 | 反直觉结论 |
|------|------|-----------|
| 穷人应该省钱 | 省钱=压缩投资自己的预算=更难跳出贫困 | 越穷越要花钱学最贵的知识 |
| 应该做职业规划 | 规划建立在「一切不变」的假设上=容错率低 | 永远别做职业规划，做人生规划 |
| 复利是最强财富工具 | 复利公式对，但现实中不存在稳定的复利条件 | 复利是金融骗局 |
| 年轻人应该躺平 | 在日本欧洲应该（机会少），在中国不应该（机会多） | 在最好的时代躺平是最差的生存策略 |
| 短视频要学各种技能 | 补短板永远补不完 | 换一个基本盘大的选题，立竿见影 |

**触发条件**：当他看到一个被广泛接受的「常识」，他自动触发反直觉推导——不是因为「喜欢抬杠」，而是因为他的操作系统底层写死了一条：「看到表面规则，立刻找实际规则」。
