# IF v2.0：认知外推引擎设计

> 基于心理学、计算机科学、哲学三领域专家的交叉讨论，设计 IF 的核心升级——让已提取的人格不仅能「回答已知问题」，还能「用自己的一套认知框架消化新知识」。

---

## 一、专家共识的核心结论

### 哲学家说（最重要的）

> **不存在「乔布斯会怎么想AI」的正确答​​案。** 即使乔布斯真的活到今天学了AI，他可能产生多种互相矛盾的看法——取决于那天早上喝了什么咖啡。

所以 IF v2.0 不追求「正确答案」，只追求 **「合理的乔布斯式推断」**。两个东西区分开：
- ❌ 「乔布斯会认为AI是X」→ 这是假的，没人知道
- ✅ 「如果乔布斯用他已知的思维框架去理解AI，他可能会沿着以下几条路径思考...」→ 这是诚实的

### 计算机科学家说（最实操的）

```
新信息 → [FRAME:本体论映射] → [REASON:束搜索] → [VERIFY:多维验证] → [STYLE:风格化] → 输出
                                       ↑                    ↓
                                       └── 反馈校准 ←────────┘
```

核心思路：不是让 LLM 自由发挥，是用**从语料里提取的认知参数作为结构化约束**，卡在 LLM 的推理路径上。

### 心理学家说（最底层的）

LLM 本身已经是粗糙的「认知外推引擎」。不需要从零造一个人类认知模型。要做的只是：从人物语料里显式提取出认知参数（类比库、论证模板、价值排序、决策启发式），然后把这些参数作为生成约束。

## 二、IF v2.0 架构

### 核心理念：从「人格模拟」到「认知约束推理」

| | IF v1.0（当前） | IF v2.0（升级） |
|---|---|---|
| 能回答什么 | 此人已知立场的问题 | **此人从未接触过的新领域** |
| 输出形式 | 单一判断 | **多条可能路径 + 置信度 + 推理链** |
| 角色定位 | 「我就是他」 | **「我模拟了他的思维框架，以下是可能的思考路径」** |
| 诚实度 | 诚实边界是静态声明 | **每个输出都自带不确定性标注** |

### 关键模块说明

**FRAME（框架映射）**：把新知识映射到此人的本体论体系。
- 直接映射：新知识中的概念 → 此人已有的概念
- 桥接映射：通过中间抽象概念桥接
- 临时类别：对全新概念创建临时类别，但打上「临时」标签

**REASON（束搜索推理）**：不是一条推理链，是**多条**。
- 生成3-5条不同推理路径
- 每条路径有不同的核心假设
- 评分：0.35×价值对齐 + 0.25×因果一致性 + 0.25×启发式忠实度 + 0.15×推理自洽

**VERIFY（多维验证）**：
- 内部连贯性：不与核心信念矛盾
- 风格一致性：风格向量在2σ内
- 反事实鲁棒性：微调输入，结论应稳定

**UNCERTAINTY（不确定性标注）**：IF v2.0 的核心创新。
- 每条输出标注推理路径编号
- 标注核心假设（「这一步假设乔布斯会把AI类比为个人计算机而非互联网」）
- 标注置信度（高/中/低，以及为什么）
- 标注「已知的未知」（「此处乔布斯生前无直接表态，纯属外推」）

## 三、输出格式（示例）

当用户问「乔布斯会怎么看AI大模型？」时，IF v2.0 的输出：

```markdown
## 乔布斯式推断：对AI大模型的可能看法

⚠️ 乔布斯于2011年去世，从未接触过当代AI。以下基于他的思维框架外推，
非事实陈述。存在多个可能路径。

### 路径1：作为「心灵的自行车」（置信度：中高）
- 核心假设：他会将LLM类比为「计算机是心灵的自行车」——不是替代人，是放大人
- 推理链：LLM放大认知能力 → 类似Mac放大创造力 → 这是好事
- 可能结论：LLM是自Mac以来最重要的「自行车」，苹果应该全力投入

### 路径2：作为「封闭系统的武器」（置信度：中）
- 核心假设：他会将LLM视为构建封闭体验的工具
- 推理链：LLM需要硬件+软件+服务垂直整合 → 这是苹果的基因
- 可能结论：本地化、隐私保护的AI才是「真正的AI」

### 路径3：作为「新的Adobe Flash」（置信度：低）
- 核心假设：他会关注LLM的不可控性和质量问题
- 推理链：不可控输出 → 糟糕的用户体验 → 像Flash一样该被淘汰
- 可能结论：当前LLM不值得认真对待，除非做到可控

### 未知领域
- 乔布斯对「涌现智能」无已知立场。此维度纯属外推，置信度低。
- 他对开源 vs 闭源的AI策略无已知参考。推测基于他对端到端控制的偏好。
```

## 四、诚实原则（哲学家强行写入）

1. **不声称唯一答案**：永远展示多条可能路径
2. **区分「已知」和「外推」**：引用此人实际言论的，标注「已知立场」。纯推论的，标注「外推」
3. **标注核心假设**：每条推理链的关键假设写在明面上，让用户判断是否合理
4. **承认无解**：遇到完全无法外推的领域（范式级新知），直接说「此人无已知参照，无法合理外推」
5. **不是「他」是新的人**：明确告诉用户，学习后的系统是此人的思维框架在新知识上的投影，不是他本人——是「用他的镜片看新世界」，不是「他复活了」
